How it works

Each share is derived in-browser, wrapped with post-quantum armor, and written to volatile storage.

01

Derive the master key

Password material never leaves the browser. Argon2id burns memory and time before any payload is sealed.

02

Generate post-quantum armor

A deterministic ML-KEM-768 keypair is derived locally and used to wrap a random 256-bit data key.

03

Seal the payload

The encrypted envelope lands on the server as ciphertext only, with one-view semantics and zero retention.

Specs: PIM=100, Iterations=4, Memory=64MB, Parallelism=1.

reiven --help
